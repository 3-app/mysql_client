MySQL Client 技术支持

适用于 iPhone 和 iPad 版 MySQL Client

如果你在连接数据库、执行 SQL、查看表结构、导出 CSV 或恢复内购时遇到问题，可以参考本页面的排查步骤。

常见问题

1. 无法连接 MySQL 数据库

请确认以下项目：

  • 主机地址、端口、数据库名称、用户名和密码填写正确。
  • 你的 iPhone 或 iPad 当前网络可以访问目标 MySQL 服务器。
  • MySQL 服务器已允许远程连接，不只监听 127.0.0.1。
  • 防火墙、安全组或路由规则已放行 MySQL 端口，通常为 3306。
  • 数据库账号拥有从当前网络来源登录的权限。

2. 提示需要安全连接认证

如果服务器或账号要求安全连接，请在连接信息中开启 SSL/TLS。若开启后仍无法连接，请确认 MySQL 服务器已正确配置 TLS 证书，并且服务器地址与证书配置匹配。

3. 查询没有返回结果

请确认 SQL 语句可以在目标数据库中正常执行，并检查当前连接的数据库是否正确。对于较大的查询，建议添加 LIMIT 限制返回行数。

4. 无法看到表列表或表结构

App 会通过 information_schema 查询数据库表、字段和索引信息。请确认当前数据库账号拥有访问相关元数据的权限。

5. Pro 购买或恢复失败

Pro 功能由 Apple App Store 内购处理。请确认：

  • 设备已登录可用的 Apple ID。
  • 网络可以访问 App Store 服务。
  • 如果你已购买，请在 App 内使用“恢复购买”。

使用建议

  • 优先使用只读或最小权限数据库账号。
  • 连接生产数据库时，建议开启 SSL/TLS。
  • 执行修改或删除数据的 SQL 前，请仔细确认语句和当前数据库。
  • 避免在公共 Wi-Fi 下连接未加密的数据库服务。

联系我们

如果问题仍未解决，请通过 App Store 中显示的开发者联系方式与我们联系。反馈时建议提供以下信息，便于定位问题：

  • 设备型号和 iOS/iPadOS 版本。
  • App 版本号。
  • 错误提示截图或完整错误文本。
  • 问题发生在哪一步，例如连接、查询、浏览表结构、导出或恢复购买。
